## 1. 节点弹出动画验证

### 1.1 功能说明
当家族树新增成员时，新节点从父节点位置弹出，带有弹性动画效果：
- 从父节点位置开始
- 使用弹性缓动动画
- 连接线逐渐绘制
- 动画时长约 1 秒

### 1.2 验证步骤

#### 步骤 1：进入演示页面
1. 启动应用
2. 导航到动画演示页面（animation-demo）

#### 步骤 2：触发节点弹出动画
1. 找到"节点弹出动画"区域
2. 点击"演示节点弹出"按钮
3. 观察动画效果

#### 步骤 3：验证动画细节
1. **起点位置**：新节点应从父节点位置开始
2. **弹性效果**：检查节点是否有弹性回弹
3. **连接线**：检查是否有连接线逐渐绘制
4. **完成回调**：动画结束后是否正确触发回调

### 预期结果
- ✅ 节点从起点位置平滑弹出
- ✅ 弹性动画自然流畅
- ✅ 连接线绘制流畅
- ✅ 动画时长适中（约 1 秒）

## 4. 性能评估

### 4.1 性能指标
使用 react-native-reanimated 实现的动画应达到以下性能指标：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| FPS | 60 | 流畅度 |
| 帧时间 | < 17ms | 每帧耗时 |
| 丢帧 | < 5% | 性能下降时丢帧比例 |
| 内存 | < 100MB | 动画期间内存占用 |

### 4.2 性能测试方法

#### 方法 1：使用 React Native Debugger
1. 打开 React Native Debugger
2. 启用 FPS 计数器
3. 触发各种动画
4. 观察 FPS 变化

#### 方法 2：使用 Flipper（推荐）
1. 安装 Flipper
2. 连接设备
3. 使用 Performance 插件
4. 监控 FPS 和 CPU 使用率

#### 方法 3：使用 Expo Doctor
```bash
npx expo-doctor
```

### 4.3 性能优化建议

#### 花瓣飘落动画
- **减少粒子数量**：在性能较弱的设备上，将数量从 20 减少到 10
- **简化计算**：减少随机摆动幅度
- **使用 `useNativeDriver`**：确保动画在 UI 线程运行

#### 血缘连线动画
- **限制点数**：建议最多 10 个点
- **简化路径**：使用更少的贝塞尔曲线控制点
- **预计算路径**：避免在动画中实时计算

#### 节点弹出动画
- **调整弹性参数**：damping=8, stiffness=100 为最佳平衡
- **限制动画数量**：同时弹出的节点不超过 5 个

### 4.4 性能检查清单

#### 基础性能
- [ ] 花瓣飘落保持 60fps
- [ ] 血缘连线保持 60fps
- [ ] 节点弹出保持 60fps
- [ ] 帧时间稳定在 17ms 以下

#### 内存使用
- [ ] 动画期间内存不剧烈增长
- [ ] 动画结束后内存正确释放
- [ ] 无内存泄漏

#### CPU 使用
- [ ] 动画期间 CPU 占用合理
- [ ] 无过度计算
- [ ] 无主线程阻塞

## 常见问题

### Q: 动画出现卡顿怎么办？
A:
1. 检查是否使用了 `useNativeDriver: true`
2. 减少花瓣数量
3. 简化血缘连线的点数
4. 检查是否有大量的 JS 计算阻塞主线程

### Q: 花瓣飘落位置不正确？
A:
1. 检查花瓣的初始 X 坐标是否正确
2. 确保使用了正确的屏幕尺寸
3. 检查 `SCREEN_WIDTH` 和 `SCREEN_HEIGHT` 是否正确

### Q: 血缘连线动画无法触发？
A:
1. 检查 `visible` 属性是否为 `true`
2. 确保 `points` 数组至少有 2 个点
3. 检查 `onComplete` 回调是否正确定义

### Q: 动画结束后出现残留？
A:
1. 确保在 `onComplete` 回调中正确设置了 `visible: false`
2. 检查组件是否正确卸载
3. 使用 React DevTools 检查组件状态

### B. 动画配置参数

#### 花瓣飘落
```typescript
{
  visible: boolean;        // 是否显示
  emoji?: string;          // 飘落 emoji，默认 🌸
  count?: number;          // 数量，默认 20
  duration?: number;       // 持续时间，默认 8000ms
  onComplete?: () => void; // 完成回调
}
```

#### 血缘连线
```typescript
{
  visible: boolean;                    // 是否显示
  points: Array<{ x: number; y: number }>; // 路径点
  color?: string;                       // 颜色
  width?: number;                       // 线宽，默认 3
  onComplete?: () => void;               // 完成回调
}
```

#### 节点弹出
```typescript
{
  visible: boolean;     // 是否显示
  parentX?: number;     // 父节点 X
  parentY?: number;     // 父节点 Y
  childX: number;       // 子节点 X
  childY: number;       // 子节点 Y
  onComplete?: () => void; // 完成回调
  duration?: number;    // 持续时间，默认 1000ms
}
```
