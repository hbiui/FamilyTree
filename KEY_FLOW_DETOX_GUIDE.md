# 关键流程 Detox 端到端测试运行指南
===================================

## 概述

本指南提供 `key-flow.spec.ts` 测试的完整运行命令和预期结果，涵盖用户指定的关键流程：
- 注册/登录
- 创建新家族（张氏家族）
- 添加成员（张三）
- 设置配偶关系（李四）
- 查看家族树，验证节点存在
- 导出 GEDCOM
- 邀请协作者

## 4. 配置项目

```bash
# 如果是第一次运行，需要预构建原生项目

```

确保以下文件配置正确：

## 5. 构建应用

### 5.1 构建 iOS 应用（仅 macOS）

```bash
# 预构建 iOS 项目
npx expo prebuild --platform ios

# 使用 Detox 构建
npm run detox:build:ios
```

**预期输出**：
```
✅  Building FamilyTree.app...
   ✓  Built successfully
   📦  Bundle size: ~25MB
```

### 5.2 构建 Android 应用

```bash
# 预构建 Android 项目
npx expo prebuild --platform android

# 使用 Detox 构建
npm run detox:build:android
```

**预期输出**：
```
✅  Building app-debug.apk...
   ✓  Built successfully
   📦  APK size: ~35MB
```

## 6. 运行测试

### 6.1 启动模拟器/设备

#### iOS 模拟器（macOS）

```bash
# 启动 iOS 模拟器
open -a Simulator
```

#### Android 模拟器

```bash
# 启动 Android 模拟器
emulator -avd Pixel_5_API_30
# 或使用 Android Studio 的 AVD Manager
```

### 6.2 运行关键流程测试

#### 方式一：使用 npm 脚本

```bash
# 运行关键流程测试
npm run detox:test -- e2e/key-flow.spec.ts
```

#### 方式二：直接使用 detox 命令

```bash
# iOS 方式
npx detox test --configuration ios.simulator e2e/key-flow.spec.ts

# Android 方式
npx detox test --configuration android.emu e2e/key-flow.spec.ts
```

### 6.3 调试模式运行

```bash
# 启用调试模式
npm run detox:test -- e2e/key-flow.spec.ts -- --debug
```

### 6.4 运行单个测试用例

```bash
# 只运行特定阶段的测试
npm run detox:test -- e2e/key-flow.spec.ts -- --testNamePattern="K1-01"
```

## 8. 故障排除

### 8.1 常见问题

#### 问题 1：找不到 ios/android 目录

**原因**：Expo 项目需要预构建

**解决方案**：
```bash
# 预构建原生项目
npx expo prebuild
```

#### 问题 2：构建失败

**解决方案**：
```bash
# 清理并重新构建
npm run detox:clean

# 重新构建
npm run detox:build:ios
# 或
npm run detox:build:android
```

#### 问题 3：找不到模拟器

**iOS**：
```bash
# 列出可用的 iOS 模拟器
xcrun simctl list devices available
```

**Android**：
```bash
# 列出可用的 Android 设备
adb devices
```

#### 问题 4：测试超时

**解决方案**：
在 `.detoxrc.js 中增加超时时间：
```javascript
module.exports = {
  // ...
  testTimeout: 300000, // 5 分钟
};
```

### 8.2 查看日志和截图

```bash
# 测试失败时，查看 artifacts 目录
ls -la e2e/artifacts/
```
