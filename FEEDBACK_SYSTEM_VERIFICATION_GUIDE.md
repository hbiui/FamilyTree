## 1. 功能概述

### 1.1 核心功能

| 功能 | 描述 | 状态 |
|------|------|------|
| 心情评分弹窗 | 用户使用 3 天后首次弹出 | ✅ 已实现 |
| 评分引导文案 | 友好的评分邀请 | ✅ 已实现 |
| 反馈表单 | 问题反馈、功能建议 | ✅ 已实现 |
| 本地存储 | 评分记录持久化 | ✅ 已实现 |
| 不再弹出 | 已评分后不再触发 | ✅ 已实现 |

### 1.2 触发规则

- **触发时机**：用户首次使用应用后第 3 天
- **触发条件**：
  1. `firstUseDate` 已记录
  2. `moodRatingGiven` 为 false
  3. 距首次使用已满 3 天
- **不再触发**：用户评分后 `moodRatingGiven` 设为 true

## 3. 首次触发条件模拟

### 3.1 触发条件检查

评分弹窗的触发需要同时满足以下条件：

1. ✅ `firstUseDate` 已设置（首次使用日期）
2. ✅ `moodRatingGiven` 为 false（未评过分）
3. ✅ 距首次使用已满 3 天

### 3.2 开发调试方法

#### 方法 1：使用开发工具重置（推荐）

1. 进入"反馈中心"页面
2. 找到"开发工具"区域
3. 点击"重置首次使用时间"

这样可以将 `firstUseDate` 设置为 4 天前，自动满足触发条件。

#### 方法 2：手动修改存储

打开 React Native Debugger 或 Flipper：

1. 找到 `feedback-storage` 存储
2. 修改 `firstUseDate` 为 4 天前的日期：
   ```json
   {
     "firstUseDate": "2024-01-01T00:00:00.000Z",
     "moodRatingGiven": false
   }
   ```
3. 重启应用

#### 方法 3：代码层面测试

在 `FeedbackProvider.tsx` 中临时修改触发天数：

```typescript
// 将 triggerDays 从 3 改为 0
<FeedbackProvider triggerDays={0}>
```

### 3.3 验证流程

```
首次使用日期检查
       ↓
  是否已过 3 天？
   ↙         ↘
  是          否
   ↓          ↓
检查评分状态    不弹出
   ↓
是否已评分？
 ↙       ↘
 是        否
 ↓        ↓
不再弹出   显示弹窗
```

### 3.4 触发时间线

| 时间点 | 行为 |
|--------|------|
| 第 0 天 | 首次使用，记录 `firstUseDate` |
| 第 1-2 天 | 不弹出（未满 3 天） |
| 第 3 天 | 首次满足条件，弹出评分 |
| 评分后 | `moodRatingGiven = true`，不再弹出 |

## 4. 本地存储记录验证

### 4.1 存储结构

评分和反馈数据存储在 `feedback-storage` 中：

```typescript
interface FeedbackState {
  firstUseDate: string | null;        // ISO 日期字符串
  moodRatingGiven: boolean;           // 是否已评分
  lastMoodRatingDate: string | null; // 最后评分时间
  totalMoodRatings: number;          // 评分次数
  feedbackEntries: FeedbackEntry[];   // 所有反馈记录
  hasSeenOnboarding: boolean;        // 是否看过引导
}
```

### 4.2 查看存储数据

1. 打开 React Native Debugger（按 `Cmd/Ctrl + D`）
2. 进入"Storage"或"AsyncStorage"标签
3. 查找 `feedback-storage` key
4. 查看存储的数据

#### 使用 Flipper

1. 连接设备到 Flipper
2. 打开"Database"插件
3. 查找 `feedback-storage`
4. 查看 JSON 数据

## 6. 常见问题

### Q: 为什么弹窗没有弹出？

**可能原因**：

1. **未满足触发条件**
   - 检查 `firstUseDate` 是否设置
   - 确认距首次使用已满 3 天

2. **已评过分**
   - 检查 `moodRatingGiven` 是否为 true
   - 评分后不会再次弹出

3. **存储未正确初始化**
   - 检查是否有控制台错误
   - 确认 Zustand 持久化配置正确

**解决方案**：

```typescript
// 在控制台执行
useFeedbackStore.getState().resetFeedback();
```

### Q: 如何强制显示弹窗进行测试？

**方法 1**：重置首次使用时间

```typescript
// 在控制台执行
const state = useFeedbackStore.getState();
state.initializeFirstUse();
state.resetFeedback(); // 清空评分状态
```

**方法 2**：修改触发天数

```typescript
// 在 FeedbackProvider 中
<FeedbackProvider triggerDays={0}>
```

### Q: 评分后想重新测试？

**方法**：重置反馈状态

1. 进入"反馈中心"
2. 找到"开发工具"
3. 点击"重置所有反馈数据"
4. 重启应用

### Q: 存储数据丢失？

**可能原因**：

1. 应用数据被清除
2. 存储格式损坏
3. 持久化配置错误

**解决方案**：

1. 检查 `zustand` 持久化配置
2. 确认存储 key 正确
3. 查看控制台错误日志
