### 1.2 验证步骤

#### iOS (VoiceOver)
1. 打开「设置」→「辅助功能」→「VoiceOver」
2. 启用 VoiceOver
3. 打开家族树应用
4. 用手指滑动浏览界面
5. 检查每个元素是否正确朗读

**朗读示例：**
```
添加成员，按钮
姓名：张三
男，出生日期：1990年1月1日
配偶：李四
子女：2个子女
双击查看详情，长按执行操作
```

#### Android (TalkBack)
1. 打开「设置」→「辅助功能」→「 TalkBack」
2. 启用 TalkBack
3. 打开家族树应用
4. 用手指滑动浏览界面
5. 检查每个元素是否正确朗读

**朗读示例：**
```
按钮：添加新成员
编辑按钮：编辑当前成员
复选框：记住密码，未选中
```

### 1.3 无障碍属性代码示例

```typescript
// ✅ 正确示例
<TouchableOpacity
  accessible={true}
  accessibilityLabel="添加新成员"
  accessibilityRole="button"
  accessibilityHint="双击添加新家族成员"
>
  <Text>+ 添加成员</Text>
</TouchableOpacity>

// ❌ 错误示例
<TouchableOpacity>
  <Text>+ 添加成员</Text>
</TouchableOpacity>
```

## 6. 常见问题排查

### 问题 1：VoiceOver/TalkBack 未朗读内容

**可能原因：**
- 元素未设置 `accessible={true}`
- 元素在屏幕外
- 元素被 `display: none` 隐藏

**解决方案：**
```typescript
// ✅ 确保元素可访问
<View accessible={true} accessibilityLabel="内容">
```

### 问题 2：朗读顺序混乱

**可能原因：**
- 组件层级结构问题
- 使用了绝对定位导致顺序错乱

**解决方案：**
- 检查组件层级
- 使用 `accessibilityElements` 控制顺序

### 问题 3：多语言切换后文案未更新

**可能原因：**
- 使用了硬编码文案
- 未正确使用 `useTranslation` hook

**解决方案：**
```typescript
// ✅ 正确使用翻译
const { t } = useTranslation();
<Text>{t('member.addMember')}</Text>

// ❌ 硬编码文案
<Text>添加成员</Text>
```

### 问题 4：关系称谓显示不一致

**可能原因：**
- 关系计算结果未使用翻译
- 直接硬编码中文称谓

**解决方案：**
```typescript
// ✅ 使用翻译映射
const relationLabels = {
  father: t('member.father'),
  mother: t('member.mother'),
  spouse: t('member.spouse'),
};
```
