# 离线模式功能验证指南

## 📋 概述

本文档提供了完整的离线模式功能验证方案，包括：
- 数据回显验证
- 队列同步验证
- 冲突处理验证

## 🎯 功能概览

### 已实现的功能

1. **网络状态管理** - 实时监听网络状态变化
2. **本地缓存存储** - 使用 MMKV 缓存家族树和最近浏览的成员
3. **离线操作队列** - 暂存离线时的编辑操作，恢复网络后自动同步
4. **UI 横幅提示** - 显示离线模式和待同步操作提示
5. **开发调试工具** - 方便测试的调试面板

### 核心架构

```
OfflineModeProvider
├── OfflineBanner (UI 提示)
├── NetworkStore (网络状态)
├── CacheStore (本地缓存)
└── OfflineQueueStore (操作队列)
```

## 🔍 验证方案

### 验证 1：数据回显验证

#### 目标
验证在离线模式下，已浏览的内容仍然可以正常显示。

#### 前置条件
- 应用正常安装并启动
- 至少有 1 个家族和 3 个成员数据

#### 测试步骤

**步骤 1：在线浏览数据**
1. 确保设备连接网络
2. 打开应用，进入「家族树」页面
3. 浏览至少 3 个成员的详情页面
4. 进入「成员」列表页面，浏览所有成员
5. 记录浏览的数据内容

**步骤 2：切换到离线模式**
- **方法 A（真机）**：开启飞行模式
- **方法 B（模拟器）**：在设置页面，点击「开发调试」→「模拟离线」
3. 验证顶部显示红色「离线模式」横幅

**步骤 3：验证数据回显**
1. 关闭应用，重新打开
2. 进入「家族树」页面 - 验证：
   - 家族树数据正常显示
   - 成员卡片正常渲染
   - 头像和基本信息可见
3. 进入「成员」页面 - 验证：
   - 成员列表完整显示
   - 搜索和筛选功能可用（基于缓存）
4. 进入之前浏览的成员详情 - 验证：
   - 成员详细信息完整显示
   - 头像、生日等信息正常

**预期结果**
- ✅ 离线模式横幅正确显示
- ✅ 之前浏览的数据完整回显
- ✅ 页面渲染流畅，无白屏或错误

---

### 验证 2：队列同步验证

#### 目标
验证离线时的编辑操作能够正确暂存，网络恢复后自动同步。

#### 前置条件
- 已完成「数据回显验证」
- 当前处于离线模式

#### 测试步骤

**步骤 1：创建离线操作**
1. 在离线模式下，进行以下操作（注意：需要在实际页面中调用队列 API）：
   - 添加 1 个新成员
   - 更新 1 个现有成员的信息
2. 验证：
   - 操作成功完成（UI 反馈）
   - 顶部横幅显示「X 个操作待同步」
   - 在设置页面的调试面板中，队列数量增加

**步骤 2：查看队列状态**
1. 进入「设置」→「开发调试」
2. 查看「离线队列」部分 - 验证：
   - 显示待同步操作数量
   - 每个操作的类型和时间戳

**步骤 3：恢复网络并同步**
1. 恢复网络连接（关闭飞行模式，或在调试面板点击「模拟在线」）
2. 验证：
   - 顶部横幅颜色变为橙色/主题色
   - 显示「X 个操作待同步」和「立即同步」按钮
3. 点击「立即同步」或等待自动同步 - 验证：
   - 显示「同步中...」
   - 同步完成后，待同步数量清零
   - 顶部横幅消失

**预期结果**
- ✅ 离线操作成功暂存
- ✅ 网络恢复后自动或手动同步
- ✅ 同步完成后队列清空

---

### 验证 3：冲突提示验证（高级功能）

#### 目标
验证在离线期间服务器数据发生变化时，能够正确处理冲突。

#### 前置条件
- 已完成「队列同步验证」
- 有能力修改后端数据（或使用模拟）

#### 测试步骤

**步骤 1：离线修改**
1. 记录当前成员 A 的信息
2. 切换到离线模式
3. 修改成员 A 的姓名为「离线修改」
4. 保持离线状态

**步骤 2：模拟服务器端修改**
1. 在另一个设备或直接修改数据库
2. 将同一成员 A 的姓名改为「服务器修改」
3. 保存服务器端修改

**步骤 3：恢复网络并同步**
1. 恢复网络连接
2. 触发同步操作
3. 观察冲突处理 - 验证：
   - 检测到冲突提示
   - 显示对比信息（本地 vs 服务器）
   - 提供选择：保留本地/采用服务器/手动合并
4. 选择「保留本地」并确认
5. 验证最终数据为「离线修改」

**预期结果**
- ✅ 冲突被正确检测
- ✅ 用户收到明确的冲突提示
- ✅ 选择后正确应用结果

## 🛠️ 调试工具使用

### 访问调试面板
1. 进入「设置」页面
2. 滚动到底部（开发模式下可见）
3. 找到「开发调试」部分

### 调试功能说明

#### 网络状态控制
- **模拟离线**：强制应用进入离线状态
- **模拟在线**：恢复到在线状态
- 实时显示网络类型和连接状态

#### 缓存管理
- 查看当前缓存的家族和成员数量
- 查看上次同步时间
- 「清除所有缓存和队列」按钮：重置所有离线数据

#### 队列管理
- 查看待同步操作数量
- 添加测试操作到队列
- 手动触发同步
- 查看同步状态

## 📊 性能指标

### 缓存性能
- **启动加载时间**：< 1 秒（从 MMKV 读取缓存）
- **内存占用**：缓存 200 个成员时增加 < 10MB
- **存储占用**：单个成员数据约 1-2KB

### 同步性能
- **队列加载**：< 100ms（100 个操作）
- **单操作同步**：< 500ms（网络正常）
- **批量同步**：100 个操作 < 30 秒

## 🐛 常见问题排查

### 问题 1：离线横幅不显示
**排查步骤**
1. 确认网络状态监听是否初始化
2. 检查 `useNetworkStore` 状态是否正确更新
3. 查看控制台是否有错误信息

**解决方案**
- 重启应用
- 在调试面板手动切换网络状态

### 问题 2：数据没有缓存
**排查步骤**
1. 确认 `OfflineModeProvider` 正确包裹了应用
2. 检查成员/家族是否成功加载
3. 查看 MMKV 中是否有数据

**解决方案**
- 在调试面板查看缓存统计
- 确保数据加载后有调用缓存 API

### 问题 3：同步失败
**排查步骤**
1. 检查网络连接是否正常
2. 查看队列操作的错误信息
3. 确认同步 API 端点是否可用

**解决方案**
- 在调试面板查看 `syncError`
- 检查网络连接后重试
